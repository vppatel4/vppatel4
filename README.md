## About Me

I'm a software engineer who works with frontend, backend and also the infrastructure underneath. Most of what I build is full-stack web apps with REST APIs, relational schemas and auth, deployed in containers; some of it leans into AI when the problem actually calls for it (retrieval over vector search, knowledge graphs) and some is data plumbing; ETL over hundreds of thousands of records. I try to write code that someone else can read, test and deploy without me in the room. Currently doing my M.S. in Computer Software Engineering at Arizona State University and building production software for public-sector clients at ASU's AI Cloud Innovation Center.

## 🚀 Featured Projects

### 🔎 [LitGraph](https://github.com/vppatel4/SER594-Team19-LitGraph)
> AI research workspace for cited cross-paper Q&A and knowledge-graph generation

LitGraph turns a pile of research PDFs into a queryable workspace: it parses and chunks papers, embeds them into a pgvector store and answers questions across the whole corpus with inline citations back to the source passages. On top of retrieval, it generates a knowledge graph linking concepts across papers, so you can see how ideas connect rather than reading each one in isolation. Built on a FastAPI backend with a React frontend, with retrieval quality validated through an automated evaluation set.

`Python` `RAG` `pgvector` `OpenAI` `FastAPI` `React` `Knowledge Graphs`

---

### 🗂️ [SyncSpace](https://github.com/vppatel4/SyncSpace)
> Full-stack project-management platform with Kanban, heatmaps, and burndown charts

SyncSpace is a team project-management tool covering the full workflow: Kanban boards with drag-and-drop, workload heatmaps to spot over-allocated members and burndown charts for tracking sprint progress. The backend exposes 20+ REST APIs with role-based authentication, wired end-to-end to a Next.js frontend with Redux/RTK Query for state and data fetching. Persistence is handled through PostgreSQL with Prisma ORM.

`Next.js` `TypeScript` `PostgreSQL` `Prisma` `Redux` `REST APIs`

### 🧭 [LifeOps Copilot](https://github.com/ashah5123/LifeOps-Copilot)
> AI student-life dashboard pulling insights across inbox, calendar, and budget

LifeOps Copilot is a single dashboard that connects a student's scattered tools- email, calendar and budget, and surfaces AI-generated insights across all of them. It authenticates via Gmail OAuth and routes context through Vertex AI (Gemini) to produce summaries and suggestions per module. The full frontend is built in Next.js, TypeScript, Tailwind and Zustand and the whole app was containerized and deployed on Google Cloud Run.

`Next.js` `TypeScript` `Vertex AI` `Gmail OAuth` `GCP` `Cloud Run`

### 🧠 [JVMLang](https://github.com/vshah113-asu/SER502-JVMLang-Team5)
> An explainable programming language — a full compiler and interpreter built in Prolog

JVMLang is a custom imperative language with a complete pipeline written in SWI-Prolog: a tokenizer, a Definite Clause Grammar (DCG) parser, AST generation and a runtime evaluator, runnable through a single command-line script. It supports the usual constructs: int/float/bool/string types, if-else, while and for loops, blocks and full arithmetic, relational, logical and ternary operators; but its real idea is explainability. Two custom statements, `watch(x)` and `explain(expr)`, let you observe a variable's value mid-execution and walk through how an expression is evaluated step by step, so the language behaves like a learning tool rather than a black box.

`SWI-Prolog` `DCG` `Compilers` `Interpreters` `Language Design`

### 🧩 [Agile Requirements Engineering Tool](https://github.com/mmehta53/SER515)
> Web platform for Agile teams to plan releases and groom backlogs end to end

A collaborative tool that supports the full Agile workflow for both core contributors ("Pigs") and stakeholders ("Chickens"): release planning, user-story creation with acceptance criteria and priorities, backlog grooming, MVP identification and marking stories sprint-ready. It's designed as a central hub that can export stories to tools like Jira and Taiga. Built with a React (Vite) frontend and a Flask backend with role-based auth, backed by MongoDB.

`React` `Flask` `MongoDB` `JWT Auth` `Agile`

### 🌐 [FoodPriceNet](https://github.com/jpimplik/SER531-Group5)
> Semantic knowledge graph modeling how food prices move from producers to consumers

FoodPriceNet is the project behind my IEEE ICSC paper. It models the relationships between food categories, the Consumer Price Index (CPI) and the Producer Price Index (PPI) as a semantic knowledge graph, so you can ask how producer-price shifts ripple into consumer prices across the food supply chain. Built on semantic-web tech; an OWL ontology authored in Protégé, RDF triples in Turtle served from an Apache Jena Fuseki SPARQL store with a React frontend and Flask backend and the whole stack containerized via Docker Compose.

`RDF` `OWL` `SPARQL` `Protégé` `Apache Jena Fuseki` `React` `Flask`
